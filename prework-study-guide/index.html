HTML

  • The head element contains the information about thte webpage
  • The body element represents the visible content shown to the user
  • The 'p' element represents a paragraph or block of text
  • The 'ol' element represents an ordered list
  • The 'title' represents the title of the webpage
  • The base defines the base URL for a webpage
  • The line element connects an external resource to the HTML document
  • The Script element embeds client-side scripts in the HTML document
  • The charset element sets the character encoding
  • HTML elements are case sensitive
  • Child elements are elements that are nested inside other elements
  • Parent elements are the elements that hold all of the children elements together
  • The process of improving the ranking of a website in search engines is called Search Engine Optimization (SEO)
  • HTML declarations are NOT case sensitive
  • The !DOCTYPE html declaration tells the browser what document type it should expect when it is opened
  • HTML attributes define additional information about an element
  • Semantic HTML elements describe its content and the role it plays on a webpage
  • When working with visible HTML elemnents, be sure that the content is contained between an opening tag and closing tag

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and swithches to it
  • git add -A: adds ALL changes to the current working branches staging area
  • git commit -m: creates a commit for the currently staged changes
  • git pull: receives a branches modifications into the local environment (your computer)
  • git push: pushes changes we have locally made to our remote GitHub branch
  • The 'origin' keyword when used in a git command indicates the source of the pull will be in the GitHub repo
  • The 'main' keyword indicates the branch
  • Always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work
