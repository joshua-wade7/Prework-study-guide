HTML

  • The head element contains the information about thte webpage
  • The body element represents the visible content shown to the user
  • The 'p' element represents a paragraph or block of text
  • The 'ol' element represents an ordered list
  • The 'title' represents the title of the webpage
  • The base defines the base URL for a webpage
  • The line element connects an external resource to the HTML document
  • The Script element embeds client-side scripts in the HTML document
  • The charset element sets the character encoding
  • HTML elements are case sensitive
  • Child elements are elements that are nested inside other elements
  • Parent elements are the elements that hold all of the children elements together
  • The process of improving the ranking of a website in search engines is called Search Engine Optimization (SEO)
  • HTML declarations are NOT case sensitive
  • The !DOCTYPE html declaration tells the browser what document type it should expect when it is opened
  • HTML attributes define additional information about an element
  • Semantic HTML elements describe its content and the role it plays on a webpage
  • When working with visible HTML elemnents, be sure that the content is contained between an opening tag and closing tag

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • There are three types of CSS: Inline CSS, Internal CSS, and the most common and External CSS Style Sheet
  • When using external CSS, you link the file using the 'link' element in the HTML's 'head'
  • Separation of Concerns refers to separating code into different files based on use. A key design principle!
  • A selector defines the element or attributes to which the decalarations will apply
  • Declarations contain two important components: the CSS property we want to apply and the value of the property i.e. {declaration: value}
  • 'Text-align' will center a title or heading to a webpage
  • 'img' assigns property values to any image element in the markup
  • The 'rel' attribute specifies the relationship between the current document and the linked document resource
  • The 'href' attribute specifies the location (URL) of the external resource
  • The 'href' given in the prework is using a relative path. This means the location is derived from the current location (in this case the index.html). To locate the style sheet from the current location, we need to look in the assets folder, thus the 'assets/style.css'

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and swithches to it
  • git add -A: adds ALL changes to the current working branches staging area
  • git commit -m: creates a commit for the currently staged changes
  • git pull: receives a branches modifications into the local environment (your computer)
  • git push: pushes changes we have locally made to our remote GitHub branch
  • The 'origin' keyword when used in a git command indicates the source of the pull will be in the GitHub repo
  • The 'main' keyword indicates the branch
  • Always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Control flow can be interrupted by using Conditional Statements and Loops.
  • JavaScript is case sensitive, be sure to double check work if you are receiving an error code.
  • Use the 'var' keyword to declare a variable - after the keyword give the variable a unique name that will help us rememeber the reference later on.
  • Use '=' to assign the specific data that we want stored in our variable.
  • 'Strings' are contained within quotation marks. All strings are surrounded by quotation marks.
  • Each line of a JavaScript code ends with a semicolon ';'.
  • 'console.log' will output whatever we add inside the parenthesis to the console 'Chrome.dev.tools'
  • The 'if' statement is a way of introducing decision making into our code within JavaScript.
  • When 'if' is used, the code will only execute IF the statement meets a condition or is truthy.
  • The '===' is a equality operator that checks to see if two values are equal, and returns a Boolean result 'true' if the values are equal and 'false' if the values are not equal.
  • An 'array' is a single variable that is used to hold a group of data. Typically, ther are used to hold data that is related in some way.
  • Arrays are paricularly useful whenwe want to perform a tast on multiple pieces of data at once.
  • An 'index' is added when we want to work with individual items within an 'array'.
  • Indices are always in a predictable pattern starting with [0].
  • The 'for' loop is used when we have numerous items within an array.
  • Three important steps for 'for' loops.
  • 1. Starting at the beginning of an array we declare a varibale 'x' and give it a value of '0'.
  • 2. The second statement if the condition. As long as the number in the variable 'x' is less than the length (the number of items in the array) of the array, the loop will keep running i.e. 'x < shapes.length';.
  • 3. The final statement is what allows the array to iterate over each item. Since the indices follow a predictable pattern, the number will increase by '1' each time. The code 'x++' is a shortcut of writing 'x+1'.
  • A 'function' is a set of instructions that tells the computer how to perform a certain task.
  • Functions do not automatically execute but require a you to 'call' the function for it to run.
  • Define a Function - give your function a unique name followed by a parentheses '()'. You add curly brackets {} that contain the code for the task we want to perform.
